En este tutorial, aprenderemos cómo usar contratos inteligentes (*smart contracts*), cómo cambiar entre cuentas, y cómo ver el estado de una cuenta. 

## ¿Qué es Cadence?

---

Cadence es un nuevo lenguaje de programación de contratos inteligentes que se utiliza en la Blockchain Flow. Cadence introduce nuevas características en la programación de contratos inteligentes que ayudan a los desarrolladores a garantizar que su código es confiable, seguro, claro y entendible. Algunas de esas características son: 

- Seguridad de tipos y un sistema de tipos estático robusto. 
- Programación orientada a recursos, un nuevo paradigma que empareja los tipos lineaales con capacidades de objetos para crear un modelo seguro y declarativo para la propiedad digital garantizando que los recursos (y sus activos asociados) solo pueden existir en una loclización a un mismo tiempo, no pueden ser copiados, y no pueden perderse o borrarse accidentalmente. 
- Pre-condiciones y post-condiciones incorporadas en las funciones y las [transacciones](/intro/glossary/#transaction)
- La utilización de la seguridad basada en capacidades, que garantiza el control de acceso requiriendo que el acceso a los objetos se restringe exclusivamente a los propietarios de los objetos y a aquellos que tienen una referencia válida al objeto. 

## ¿Qué es el Flow Developer Playground?

---

El [Flow Playground](https://play.onflow.org) incluye un editor accesible mediante el navegador y un emulador para experimentar con Flow. Utilizando el Flow Playground, se pueden escribir contratos inteligentes en Cadence, desplegarlos a una cadena de bloques emulada de Flow, y enciar transacciones. 

El Flow Playground debería ser compatible con cualquier navegador estándar, pero recomendamos el uso de Google Chrome, dado que solo se ha probado y optimizado para ese navegador hastaa el momento.

## Conociendo el Playground

El playground contiene todo lo que se necesita para familiarizarse con el despliegue de contratos inteligentes Candence y la inteacción con Transacciones y fragmentos de código (*Scripts*).

El Playground está pre-configurado con plantillas de contratos y transacciones que corresponden con cada uno de los cinco tutoriales en esta documentación (Hello World, Fungible Tokens, NFTs, Marketplace, and Composable Resources). Para cargar los contratos de un tutorial specífico, pulsa en el enlace "Examples" en la parte superior del Playground. Eso abrirá un menú con cada tutorial. 

Cuando se pulsa en uno de esos enlaces, el tutorial se abrirá en una nueva pestaña y los contratos, transacciones y scripts se cargarán en las plantillas en el Playground para su uso.

## Cuentas y Contratos 

The Accounts section on the left side of the screen is where the active accounts are listed and selected. You can click on an account tab to view the contract that is associated with that account in the main editor.

<Img src="https://storage.googleapis.com/flow-resources/documentation-assets/cadence-tuts/playground-accounts.png" />

When you have a .cdc file open in the account editor that contains a contract, you can click the deploy button to deploy that contract to the currently selected account.

<Img src="https://storage.googleapis.com/flow-resources/documentation-assets/cadence-tuts/playground-deploy.png" />

After a few seconds, you should see a message in the console confirming that the contract was deployed. You should also see the name of the contract show up in the tab for that account, indicating the account now has that contract deployed to it.

You can also select transactions and scripts from the left selection menu and submit them to interact with your deployed smart contracts.

This is just a small set of the things you can do with the Playground. If you would like a more detailed explanation of the different Playground features, look at the Playground Manual.

## Resources

Each tutorial in this package uses several files containing transactions, contracts, and scripts. All the code you need will be provided in the text of the tutorials for you to copy and paste, or you can use the pre-generated tutorial setups in the Playground.
